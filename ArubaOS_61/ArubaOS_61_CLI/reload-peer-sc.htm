reload-peer-sc (deprecated)

reload-peer-sc

Description

This command performs a reboot of the Supervisor Card in 6000 Mobility controllers.

Syntax

No parameters.

Usage Guidelines

This command is supported only on controllers that require the Supervisor Card (SC). The SC processes all traffic from the line cards (LCs) and performs all management functions.

 

This command is not applicable to the Aruba Multi-Service Mobility Module Mark I

The reload-peer-sc command allows one SC to reset the other SC in a dual SC configuration. This does not affect the SC on which the command is executed and the LCs which it controls.

After you use the reload-peer-sc command, the controller prompts you for confirmation of this action and returns the following message:

Do you really want to reset the peer Supervisor Card(y/n):

  • Enter y to reboot the peer SC.

  • Enter n to cancel this action.

The command will timeout if you do not enter y or n.

Example

The following command reboots the peer SC:

reload-peer-sc

The controller returns the following messages:

Do you really want to reset the peer Supervisor Card(y/n):

Peer Supervisor Card will now restart.

Command History

This command was introduced in ArubaOS 1.0.
