show local-userdb-guest

show local-userdb-guest {[maximum-expiration][start <offset> page <page_size]}

Description

Shows information about guest user’s accounts uin the local user database.

Syntax

Parameter

Description

maximum-expiration

How long the account is valid, in minutes, in the internal database. 

<offset>

The guest user account record’s location (by number )as it is listed in the database.

<page_size>

The number of user account records that display on one page.

Usage Guidelines

Issue this command without any parameters to display a general overview of guest user’s accounts in the database. Use the maximum-expiration parameter to show how long the account is valid for in minutes. Use the start <offset> page <page_size> parameters to control which guest user account records in the database display initially and the number of account records displayed on a page.

Example

This example shows the basic summary of guest user accounts in the database.

The output of this command includes the following parameters

:

Parameter

Description

Name

Name of the guest user.

Password

The guest user’s password.

Role

Role for the guest user. This role takes effect when the internal database is specified in a server group profile with a server derivation rule. If there is no server derivation rule configured, then the user is assigned the default role for the authentication method.

E-mail

Shows the email address of the guest user account.

Enabled

Shows whether the account is enabled or disabled.

Expiry

Shows the expiration date for the user account. If this is not set, the account does not expire.

Status

Shows whether the profile has enabled or disabled the ability to use the HTTP protocol to redirect users to the captive portal page.

Sponsor-Name

Shows the sponsor’s name. The sponsor is the person who sponsors the guest for this visit.

Grantor-Name

Shows the grantor’s name. The grantor is the person (usually the guest provisioning user) who creates the guest on the controller.

User Entries

Shows the number of user accounts in the database.

show local-userdb-guest username verbose

show local-userdb-guest {[username <name>] [verbose]}

Description

Shows general or detailed information about a specific guest user account or all guest user accounts in the internal database.

Usage Guidelines

Issue the show local-userdb-guest username command to display information about a specific guest user account in the database. Use the verbose option with the username to display detailed information about a specific guest user account. Or, use only theverbose parameter to display detailed information about all guest user accounts in the local database.

Example

This example shows basic information of guest username Paula in the database.
